# rko-sys Bindings Design

Generate Rust FFI bindings for Linux **kernel-internal** headers using
[bnd](https://github.com/youyuanwu/bnd) (`bnd-winmd` 0.0.2).

## Pipeline

```
kernel headers ──→ libclang ──→ bnd-winmd ──→ .winmd ──→ windows-bindgen ──→ rko-sys Rust FFI
```

## Crate Layout

```
rko/
├── rko-sys/                   # Output crate (#![no_std], committed)
│   ├── Cargo.toml             # features: types, err
│   ├── src/
│   │   ├── lib.rs
│   │   ├── printk.rs          # Hand-written: _printk extern + KERN_* constants
│   │   ├── module.rs          # Hand-written: module_license! etc macros
│   │   └── rko/               # Generated by rko-sys-gen
│   │       ├── mod.rs
│   │       ├── types/mod.rs   # 119 typedefs, 10 structs
│   │       └── err/mod.rs     # 150 errno constants
│   └── winmd/
│       └── rko-sys.winmd
│
├── rko-sys-gen/               # Generator crate
│   ├── Cargo.toml             # bnd-winmd 0.0.2, windows-bindgen 0.66
│   ├── src/lib.rs             # generate() function
│   ├── src/main.rs            # CLI entry point
│   └── rko.toml               # Partition config + kernel clang args
│
├── samples/hello/             # Minimal kernel module sample
│   ├── Cargo.toml
│   └── hello.rs
│
├── linux/                     # Kernel source tree (symlink)
└── linux_bin/                 # Kernel build output (symlink)
```

## Key Files

| File | Description |
|---|---|
| `rko-sys-gen/rko.toml` | Partition config with kernel include paths (`LINUXINCLUDE` + `USERINCLUDE` order) and clang args (`-nostdinc`, `-D__KERNEL__`, `-DMODULE`, etc.) |
| `rko-sys-gen/src/lib.rs` | Two-step generate: bnd-winmd → .winmd → windows-bindgen `--package --no-toml` |
| `rko-sys/src/printk.rs` | `_printk` variadic extern + `KERN_*` byte-array constants (matching kernel bindgen output) |
| `rko-sys/src/module.rs` | `module_license!`, `module_author!`, `module_description!` macros using `#[unsafe(link_section = ".modinfo")]` |
| `samples/hello/hello.rs` | Working init/exit module with `_printk`, `.modinfo`, ELF section annotations |

## Library Name: `"kernel"`

In `rko.toml`, `library = "kernel"` is a logical name. Current partitions
(types, err) have zero functions — only typedefs and constants — so it has
no effect. Kernel symbol resolution is handled by the module loader at
`.ko` load time, not the Rust linker.

## Known Limitations

| Issue | Impact | Mitigation |
|---|---|---|
| **Variadic functions** (`printk`, …) | Auto-skipped by bnd-winmd | Hand-written extern in `printk.rs` |
| **Static inline functions** | Not visible to libclang | Wrap in C helpers compiled into a static lib |
| **Complex macros** (`IS_ERR`, `container_of`, …) | Invisible to libclang | Re-implement in Rust or wrap in C helpers |
| **`-include autoconf.h` missing** | `phys_addr_t`/`dma_addr_t` wrong size | Must inject at runtime — see `bugs/phys-addr-wrong-size.md` |
| **`typedef _Bool bool`** | Recursive type alias | Manual patch — see `bugs/bool-recursive-type-alias.md` |
| **Arch-specific** | Currently x86_64 only | Parameterize include paths later |

See `docs/design/bugs/` for full bug reports.

## Workflow

```sh
# 1. Generate rko-sys source (run once, or after header changes)
cargo run -p rko-sys-gen

# 2. Build the workspace
cargo build
```

The generated `src/rko/` and `winmd/` are committed so downstream
consumers do not need the kernel source tree or libclang.
