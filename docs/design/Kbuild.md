# Kbuild Integration Design

Build out-of-tree kernel modules (`.ko`) from Rust crates using
**cargo** for Rust compilation and **Kbuild** for final module linking.

**Status: âś… Implemented** â€” `samples/hello/hello.ko` builds end-to-end.

**Prerequisites:**
- Kernel built with `CONFIG_RUST=y` and `LLVM=1` (clang)
- `linux_bin/scripts/target.json` exists (generated by kernel build)
- Rust 1.93.0+ with `rust-src` component (`rustup component add rust-src`)

## Overview

```

cargo build (staticlib)          Kbuild (make -C LLVM=1)
        â”‚                              â”‚
  hello.rs â”€â”€â†’ libhello.a â”€â”€â†’ ld -r --whole-archive â”€â”€â†’ hello.ko
```

**Two-phase build:**

1. **Cargo phase** â€” `RUSTC_BOOTSTRAP=1 cargo build --release` compiles
   the module crate and all dependencies (`rko-sys`, `core`) into a
   single `staticlib` archive (`libhello.a`). Then `ld -r --whole-archive`
   converts it to a relocatable `.o` (needed because Kbuild's `LD -r`
   doesn't use `--whole-archive`, which would drop unreferenced sections).

2. **Kbuild phase** â€” `make -C <ksrc> O=<kout> M=<src> MO=<build> LLVM=1`
   takes the pre-built `.o`, runs modpost, compiles `.mod.c`, and links
   the final `.ko`. All build artifacts go into the `MO` directory.

### Why `staticlib` + `--whole-archive`

- **`staticlib`** bundles crate + deps + `core` into one `.a`.
  No symbol conflicts with `CONFIG_RUST=y` kernels â€” the module's
  `core` symbols are local definitions (no `__ksymtab` entries), so
  the module loader ignores them.
- **`--whole-archive`** is required because `ld -r` on a `.a` only
  pulls in members that resolve undefined references. Without it,
  `.modinfo` sections and `#[used]` statics get dropped.

### `RUSTC_BOOTSTRAP=1`

Enables nightly `-Z` flags on stable rustc. The kernel does the same
(`Makefile: export RUSTC_BOOTSTRAP := 1`). Needed for `-Zbuild-std=core`
and custom target JSON support.

### `LLVM=1`

The kernel was built with clang (`CONFIG_CC_IS_CLANG=y`). Kbuild must
use `LLVM=1` to compile `hello.mod.c` with the correct compiler flags.

## Rustc Flags

Flags in `samples/hello/cargo-kernel.toml` replicate kernel's
`KBUILD_RUSTFLAGS` from `scripts/Makefile.build`:

| Flag | Why |
|---|---|
| `-Cpanic=abort` | No unwinding in kernel context |
| `-Cno-redzone=y` | Kernel code can be interrupted; red zone is unsafe |
| `-Ccode-model=kernel` | Kernel address space (high addresses) |
| `-Crelocation-model=static` | Kernel modules are relocatable ELF, not PIC |
| `-Csymbol-mangling-version=v0` | Match kernel symbol expectations |
| `-Ccodegen-units=1` | Consistent optimization |
| `-Cforce-unwind-tables=n` | No DWARF unwind |
| `-Ctarget-feature=-sse,...,-avx2` | Kernel code must not use FPU/SIMD |
| `-Zfunction-return=thunk-extern` | Retpoline mitigation (`CONFIG_MITIGATION_RETHUNK`) |
| `-Zcf-protection=branch` | IBT/CET mitigation (`CONFIG_X86_KERNEL_IBT`) |

## Build Workflow

All builds go through CMake â€” no per-sample Makefiles:

```sh
# Configure (generates Kbuild files)
cmake -B build
cmake -B build -DENABLE_KVM=OFF    # for CI / no KVM

# Build all modules
cmake --build build

# Build one module
cmake --build build --target hello_ko

# Test all modules in QEMU
ctest --test-dir build

# Test one module
cmake --build build --target hello_ko_test

# Clean one module
cmake --build build --target hello_ko_clean
```

Or step-by-step (what CMake does internally):

```sh
# 1. Cargo builds staticlib (from samples/ workspace)
cd samples
RUSTC_BOOTSTRAP=1 cargo --config cargo-kernel.toml \
  --config 'build.target="../../linux_bin/scripts/target.json"' \
  -Z unstable-options build --release -p hello --artifact-dir hello/build

# 2. Convert .a to .o (--whole-archive preserves all sections)
ld -r --whole-archive hello/build/libhello.a -o hello/build/hello_rust.o

# 3. Kbuild produces .ko (O= kernel build, MO= module output)
make -C ../linux O=../linux_bin M=$(pwd)/hello MO=$(pwd)/hello/build LLVM=1 modules

# 4. Verify
modinfo hello/build/hello.ko   # license=GPL, author=rko, vermagic=6.19.0 ...
```

### Kbuild variables

| Variable | Description |
|---|---|
| `-C $(KSRC)` | Kernel **source** directory |
| `O=$(KOUT)` | Kernel **build output** directory (separate obj tree) |
| `M=$(CURDIR)` | Module source directory (where `Kbuild` lives) |
| `MO=$(MOUT)` | Module build output (`.o`, `.ko`, `.mod.c` go here) |
| `LLVM=1` | Use clang (kernel was built with `CONFIG_CC_IS_CLANG=y`) |

## Implementation Notes

### `.modinfo` sections use `global_asm!`

The `module_license!`, `module_author!`, `module_description!` macros
in `rko-core/src/module.rs` use `global_asm!` to emit `.modinfo` strings.

Using `#[link_section = ".modinfo"]` on Rust statics does **not** work â€”
LLVM optimizes `[u8; N]` array statics into `.rodata` references with
relocations, even when the value is a compile-time constant. modpost
expects raw `key=value\0` bytes inline in the section, not relocations.

The `global_asm!` approach emits the bytes directly via assembler
directives:
```rust
::core::arch::global_asm!(
    ".section .modinfo,\"a\"",
    concat!(".ascii \"license=", $val, "\\0\""),
    ".previous",
);
```

### Two workspaces

Samples live in a separate Cargo workspace (`samples/Cargo.toml`) with
`panic = "abort"` set in their profile. The root workspace (`Cargo.toml`)
uses the default panic strategy (unwind). This separation is necessary
because `staticlib` + `no_std` crates require `panic = "abort"`, but
applying that workspace-wide would affect the host-target generator
(`rko-sys-gen`). Samples are listed in the root workspace's `exclude`.

Both workspaces use `[workspace.dependencies]` to centralize versions.
The samples workspace declares `rko-core` as a workspace dependency, and
each sample references it via `rko-core.workspace = true`.

### Kbuild generated by CMake

Each sample needs a `Kbuild` file, but these are generated by CMake at
configure time (via `file(WRITE)` in `add_kernel_module()`). Generated
`Kbuild` files are `.gitignore`'d. Each sample has its own `CMakeLists.txt`
with a single `add_kernel_module(CHECKS ...)` call.

### objtool warnings

The build produces 2 objtool warnings from `___rust_probestack` (a
`compiler_builtins` intrinsic). The `-Zfunction-return=thunk-extern` and
`-Zcf-protection=branch` flags eliminate all warnings from our code and
`core`; only the dead `probestack` code remains.
